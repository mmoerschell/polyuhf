\chapter{Related Work}
\label{ch:related-work}

This thesis builds on polynomial hash implementation techniques, program generation for prime-field polynomial hashing, and newer multivariate polynomial hash designs.

\section{Efficient Prime-Field Polynomial Hashing}
\label{sec:sok}

Degabriele, Gilcher, Govinden, and Paterson give a systematic overview of polynomial hashing over prime fields~\cite{degabriele2024sok}. They separate high-level choices, such as the construction and finite field, from implementation choices, such as limb representation, multiplication strategy, and carry propagation. This separation is central here: the DSL describes polynomial structure, while compiler settings select the field, target, vectorization, multiplication algorithm, and carry schedule.

Their work also emphasizes delayed carry propagation and limb realignment. Unsaturated limbs leave space in each machine word to accumulate overflow bits, reducing sequential carry chains. Our compiler exposes this through carry instructions in the IR and a configurable realignment mode.

The paper also proposes prime-field designs that revisit Poly1305's choices on modern hardware.

\section{SPHGen}
\label{sec:sphgen}

SPHGen is a program generator for fast polynomial hash functions over Crandall prime fields~\cite{pegolotti2026sphgen}. It takes field and target parameters as input and emits vectorized load, add, multiply, carry, and main-loop code for the selected representation and architecture.

Our implementation uses a similar code-generation setting, but exposes it through a small DSL rather than fixed polynomial templates.

SPHGen targets a specific class of simple polynomial hash functions; this project starts from a DSL that can express several patterns.

\section{New Multivariate Polynomial Designs}
\label{sec:newdesigns}

The work on new multivariate-polynomial universal hash functions proposes two-level designs combining lower-level and higher-level hash functions~\cite{gilcher2026newdesigns}. At the lower level, it studies MMH, SQH, NMH, and HKM constructions. The definitions used as reference examples are collected in Appendix~\ref{app:polynomial-definitions}.

MMH is a sum of products. SQH replaces products by squarings. NMH groups message and key elements pairwise and is also a sum. HKM is a sequential recurrence. The paper appears to contain a typo in the recursive call; throughout this thesis we use the intended recurrence and express it iteratively using a left fold.

\section{Prime-Field Hashing Beyond Poly1305}
\label{sec:prime-field-beyond-poly1305}

The workshop paper on universal hash designs discusses why polynomial hash functions should not be tied to the exact Poly1305 parameter set~\cite{degabriele2024accordion}. Prime-field polynomial hashing remains attractive beyond \(2^{130}-5\), especially when the security level, message-length regime, or target platform changes.

This reinforces the need for tooling that can vary the field and implementation parameters without rewriting the hash function by hand.
